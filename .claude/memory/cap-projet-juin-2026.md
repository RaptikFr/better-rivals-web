Le 19/06/2026, après une série d'améliorations livrées, le proprio considère le site ET le relais comme **quasi feature-complets** : il n'a plus tellement d'idées *nouvelles* à inventer, et pense que les prochaines viendront surtout des **retours d'utilisateurs une fois le site public**.

**Why** : éviter de lui pousser des features spéculatives « pour meubler ». Le backlog à valeur est déjà défini ; le projet entre en phase finition + attente de feedback public.

**How to apply** : dans les prochaines sessions, ne pas inventer de nouvelles features non sollicitées. Se concentrer sur ce qui est concret et déjà cadré :
1. **Tests relais** = valider les offsets trace en jeu → débloque le **delta live (#1)**. Voir [[todo-proprio-delta-live]] et [[feature-trace-telemetrie]].
2. **Finir la couverture `world_records`** (script OCR, circuits restants) — tâche de données, pas de dev.
3. **Chaîne télémétrie déjà planifiée** qui découle de la trace : #1 delta live → #3 coach post-tour → #5 copilote de réglage, puis #4 régularité, #12 écuries. Voir [[roadmap-idees-juin-2026]].

Proposer du neuf est OK si ça répond à un retour public concret ou à une demande explicite du proprio — pas en remplissage.
