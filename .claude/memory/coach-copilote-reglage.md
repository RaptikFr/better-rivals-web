SmashableVelDiff=0 sur les captures (smashables = décors cassables, pas murs/bordures) → n'aide pas au filtrage bordure ; ajouté quand même à `est_suspect` (>0,5 = choc franc).

**Levier TEMPÉRATURE ajouté le 22/06** (`analyser_temp`, `diagnostic_temp`) : médiane par pneu, AV vs AR = 4e juge d'équilibre INDÉPENDANT (l'essieu le plus chaud travaille le plus). Sert de CORROBORATEUR : le conseil **antiroulis (sous-virage) sort désormais** si l'avant chauffe nettement plus (Δ>SEUIL_TEMP_DELTA=12°F, à calibrer) ET slip-angle penche sous-virage — ça débloque enfin le levier équilibre qui était mort. + détection surchauffe relative d'un pneu (>18°F au-dessus de la moyenne → pression à chaud plus basse). Intègre le freinage (avant) → corroborateur, pas juge absolu. Sur capture test : Δ+7°F → « proches » → pas de faux conseil.

**Logique d'équilibre = 3 juges** : slip-angle AV/AR (biaisé : le slip-angle AV inclut la géométrie de braquage → sur-déclare le sous-virage), contre-test lacet (C=médiane yaw/(v·steer) en bas régime), et **β = angle de dérive caisse atan2(velX,velZ)** = juge ROBUSTE (ne détecte que le survirage/glisse). Règle prudente : conseil de sous-virage seulement si slip-angle ET lacet d'accord ; survirage piloté par β.

**Fiabilisation faite le 22/06** (validée sur captures, PAS encore en jeu) : (1) filtrage des frames polluées `est_suspect()` — |ax|/|az|>30 m/s² ou slip-angle>0,6 rad = bordure/choc/tête-à-queue (~15 % des frames !) ; (2) segmentation à HYSTÉRÉSIS (entrée 4.0, sortie 2.5 m/s²) + FUSION des zones proches (un virage haché = 1 virage) ; (3) équilibre évalué sur le RÉGIME ÉTABLI (frames ≥0,7×g_max, près de l'apex) au lieu de toute la durée. Résultat : 82→52 virages, faux survirages de bordure éliminés, verdict prudent « voiture neutre ». RESTE : valider en jeu qu'une vraie voiture sous-vireuse est détectée (calibrer SEUIL_BALANCE/U).

**⚠️ Honnêteté sur l'état des leviers** (le proprio a corrigé une surestimation de ma part) : un conseil n'existe que s'il SE DÉCLENCHE vraiment, pas s'il est juste écrit dans un texte. Réellement actifs : **frein** (blocage AV/AR → balance, le seul qui sort souvent), **diff accél / survirage** (seulement si β voit une glisse), **ressorts/hauteur** (seulement via talonnage). L'**antiroulis** n'est mentionné que dans la branche « sous-virage confirmé » qui exige slip-angle ET lacet d'accord → ne sort JAMAIS (juges anti-corrélés). La pression pneus = même branche morte. Donc l'équilibre (antiroulis/ressorts) est de fait BLOQUÉ tant que les juges ne sont pas validés en jeu.

**Leviers amortisseurs + aéro AJOUTÉS le 22/06** dans coach_diag.py : 
- `analyser_amortisseurs` : compte les inversions de débattement/s par essieu (hystérésis MIN_OSC_AMPL). NIVEAU absolu NON calibré → conseil émis SEULEMENT sur déséquilibre AV/AR (>1,4×). Sur la capture test : AV 7,2 ≈ AR 7,1 → aucun conseil (bien).
- `analyser_aero` : équilibre virages lents vs rapides, découpage ADAPTATIF (médiane des vitesses de virage) + garde d'écart ≥8 m/s ; comparaison RELATIVE du ratio slip-angle (annule le biais constant). β en virage rapide = appui AR. Sur capture test : ratios 1,23 vs 1,18 → « appui cohérent ».
Les deux demandent une CALIBRATION en jeu (rouler soft/stiff dampers, plus/moins d'aileron) avant de durcir les seuils.

**Diff DÉCÉLÉRATION + pression de frein AJOUTÉS le 22/06** (`analyser_entree`, `diagnostic_entree`) : isole le trail-braking (frein>40/255 + |volant|>12 + v>8 m/s) et compte les frames en glisse arrière (β) → entrée survireuse = diff décel trop ouvert/arrière trop libre/frein trop AR. Pression de frein : si blocage AV+AR ≥30 % du freinage → réduire la pression globale. Ajouté aussi au mode `compare` (ligne « Entrée survireuse % »). Sur capture test : 104 frames trail-braking, 0 glisse → « entrée stable » (pas de faux positif).

**BOÎTE AJOUTÉE le 22/06** (`analyser_boite`, `diagnostic_boite`) : à plein gaz (accel>220), regarde le RPM dans le DERNIER rapport utilisé. Rupteur tapé (RPM≥98,5 % max) >10 % du temps → rapport final trop COURT (allonger) ; pic <90 % du rupteur → trop LONG (raccourcir, si ligne droite assez longue). Sur capture test : dernier rapport à 94 % → « cohérent », pas de faux conseil.

**Seuls leviers restants NON faisables** (confirmé par la doc officielle) : ❌ carrossage (besoin gradient de temp sur la largeur du pneu, absent), ❌ usure pneu (champ TireWear absent de FH6), ❌ pression précise (aucun champ pression ; on n'a que la temp). La temp pneu donne au mieux un proxy de pression « à chaud » par surchauffe relative. TOUS les autres sont codés : équilibre/antiroulis (bloqué calibration juges), frein balance+pression, diff accél+décél/entrée, ressorts/hauteur (talonnage), amortisseurs, aéro, boîte. **En attente de CALIBRATION en jeu** (captures de référence à comportement connu) pour durcir les seuils — sauf frein/diff/boîte qui sont déjà fiables (pilotés par β ou RPM). **Protocole détaillé** dans `C:\Users\night\OneDrive\Relais\PROTOCOLE_CALIBRATION_COPILOTE.md` (OneDrive, hors git) : captures `neutre/sousvireuse/survireuse/amorto_mou/amorto_dur/aero_min/aero_max.bin` + journal de vérité terrain à remplir.
